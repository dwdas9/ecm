# IBM - Oil & Gas Document Management
**Role:** Application Developer - ECM  
**Platform:** OpenText Documentum 6.x · OpenText Captiva 6.5

---

## Overview

Centralised document management and archiving for a major oil & gas client across EMEA, Iberia, and ASPAC. Multi-channel Captiva ingestion, Documentum storage, SAP integration, multi-vendor environment.

---

## Tech Stack

- **OpenText:** Documentum 6.x (Content Server, Composer, Webtop, DFC, BPM), Captiva 6.5 (InputAccal Server, ScanPlus, IndexPlus, Image Enhancement, Multi, ODBC Export, Documentum Export, eInput/eScan/eIndex, Dispatcher Classification & Recognition, ScaleServer)
- **Databases:** Oracle 11g, SQL Server 2008
- **Infrastructure:** Apache Tomcat 6.0, Windows Server
- **Languages:** ASP.NET, C#.NET, VBA

---

## Key Achievements

- **Multi-Channel Ingestion:** Created multi-channel Captiva processes to ingest documents from network folders, emails, and scanners
- **Advanced Processing:** Built Dispatcher Classification and Recognition workflows with OCR-based template matching and barcode recognition
- **Backend Integration:** Created SOAP-based APIs and consumed using proxy classes in .NET code module for data validation
- **Platform Upgrade:** Performed Captiva and Documentum upgrade from 5.3 to 6.5 with HA/DR using MSCS cluster and ScaleServer
- **SharePoint Infrastructure:** Architected multi-tier SharePoint farm with NLB and MSCS SQL clustering; designed HA/DR with database mirroring
- **Vendor Liaison:** Provided L3 support and served as primary liaison between client, OpenText/EMC, and third-party scanning vendors (Iron Mountain)

---

## Impact

Implemented enterprise-wide document management and capture solution supporting global oil & gas operations across three continents with centralized archiving and integrated SAP workflows.
